## Storage Types

### Types

#### Volatile

**Key:**

- Volatile
- Random Access
- Bytes
- Smaller
- Faster
- More Expensive

**Types:**

- CPU Registers
- CPU Cache
- DRAM

#### Non Volatile

**Key:**

- Non-volatile
- Sequential Access
- Blocks/Pages
- Larger
- Slower
- Cheaper

**Types:**

- SSD
- HDD
- Network/Remote Storage

### Access Times

- **L1 Cache:** 1 ns
- **L2 Cache:** 4 ns
- **DRAM:** 100 ns
- **SSD:** 16,000 ns
- **HDD:** 2,000,000 ns
- **Network/Remote Storage:** ~50,000,000 ns

### Design Goals

- Manage data that exceed the available memory
- Minimize reading/writing to disk (expensive)
- When accessing data on disk, maximize sequential access (Minimize Random Access
  )
